org.jscep.client
Class Client

java.lang.Object
  org.jscep.client.Client

public final class Client
extends Object

The Client class is used for interacting with a SCEP server.

Typical usage might look like so: 

 // Create the client
 URL server = new URL("http://jscep.org/scep/pkiclient.exe");
 CertificateVerifier verifier = new ConsoleCertificateVerifier();
 Client client = new Client(server, verifier);
 
 // Invoke operations on the client.
 client.getCaCapabilities();
Each of the operations of this class is overloaded with a profile argument to support SCEP servers with multiple (or mandatory) profile names.

Constructs a new Client instance using the provided CallbackHandler for the provided URL.

The CallbackHandler must be able to handle CertificateVerificationCallback. Unless the CallbackHandler will be used to handle additional Callbacks, users of this class are recommended to use the Client(URL, CertificateVerifier) constructor instead.

Parameters:

getCaCertificate

public CertStore getCaCertificate()
                           throws ClientException
Retrieves the certificates used by the SCEP server.

This method queries the server for the certificates it will use in a SCEP message exchange. If the SCEP server represents a single entity, only a single CA certificate will be returned. If the SCEP server supports multiple entities (for example, if it uses a separate entity for signing SCEP messages), additional RA certificates will also be returned.

Returns:
the certificate store.
Throws:
ClientException - if any client error occurs.
